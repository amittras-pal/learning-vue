Watchers.

They're like side effects performed using React.useEffect.

Use Case:

Watchers are used to run some function when one of the data property changes. Major Use Cases: (just like useEffect)

  • Setting Timers
  • HTTP calls
  • Interacting with storage
  • Rare dom interactions, like setting up mutation observers.
  • Any other external task...

They are functions with the same name as the data property they're depending on . They receive 2 arguments, newValue and oldValue

Just like React.useEffect, you should not use them to compute values and copy data between data properties.

They can also run on computed values; just name the function the same name as the computed value function.
