Conditionals & Lists.

How to render content conditionally and repeatedly.

  • v-if Directive for rendering elements conditionally.
    • It accepts any JS conditional that results in a boolean value
  • v-else Rendered when the v-if conditional is falsy.
    • Requires to placed on an immediate next sibling.
  • v-else-if Can construct an if-else-ladder tree for rendering stuff based on different conditions.
    • Requires to placed on an immediate next sibling of a v-if and before a v-else.
  • v-show Similar to v-if but instead of removing the node from the DOM when condition is false, it applies a display:none to the DOM node where it is applied.
    • Use when an element is frequently switching it's visibility state, as adding/removing elements costs performance.
    • But use sparingly since there'll be extra node with this which may not be required.
  • v-for For rendering lists. v-for="item in list"
    • Can access index. v-for="(item, index) in array"
    • Can also loop through JSON objects. v-for="(value, key, index) in object"
    • Can loop through a range of numbers. v-for="num in 100"
    • for-of loop is also supported instead of for-in loop, but it's not recommended.
    • List items should have unique keys, just like in React. This has similar implications as in React.
